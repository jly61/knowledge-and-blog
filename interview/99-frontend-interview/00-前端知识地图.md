## 0. 使用方式

这份知识地图不是用来一次性背完的，而是用来判断：

1. 我现在学到哪里了？
2. 哪些模块是企业面试高频？
3. 哪些知识需要结合项目表达？
4. 哪些知识只是了解即可？
5. 哪些可以作为中高级加分项？

建议每个知识点都按下面格式沉淀成面试卡片：

```md
## 题目

### 一句话结论

### 核心原理

### 项目场景

### 常见追问
```

掌握标准：

| 层级 | 标准 |
|---|---|
| 听过 | 看答案能看懂 |
| 初步掌握 | 不看答案能说 30 秒 |
| 面试可用 | 不看答案能讲 2 分钟 |
| 真正掌握 | 能回答 2 个以上追问 |
| 中高级表达 | 能结合项目讲方案、取舍和问题排查 |

## 2.3 企业项目中的 TS

### 重点方向

- 权限 code 类型
- 组件 props 类型
- hooks / composables 返回类型

### 面试表达重点

> TS 的价值不是多写一些类型，而是把复杂业务规则前置到开发阶段，减少运行时错误。例如流程配置、表单配置、权限配置，如果没有类型约束，很容易出现字段写错、接口返回不一致、配置漏项的问题。
